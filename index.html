Kronk

A swiss-army-knife for HTTP services.

Welcome

If you've worked with an HTTP service/API, you may have noticed how difficult it can be to troubleshoot data or performance issues. Kronk is a command line utility that gives you the tools to make this process much less painful. With Kronk, you easily parse and segregate data, run diffs between the parsed data from different queries, and easily replay logs and loadtest your HTTP applications. All parsed data is printed as expanded JSON, which makes it more readable and easy to compare. Here are a few examples of what it can do:

Display data formatted as JSON

$ kronk host.com/path.plist

Diff response data from two queries

$ kronk host.com/path.xml host.com/path.json

Parse from STDIN

$ curl host.com/path.json | kronk --parser json

Replay GET logs against any host

$ cat access.log | grep GET | kronk stage-host.com --replay --qps 100

Select specific data to display

# Return only name and text fields of an array of tweets.
$ kronk host.com/path.json -- /tweets/*/name /tweets/*/text

Kronk is capable of much more. Read more about it on the WIKI. The latest stable Kronk release is also available from RubyGems.
